# Code Review Skill

## Purpose
This skill helps you perform comprehensive code reviews.

## Steps
1. Read the code carefully
2. Check for bugs and edge cases
3. Look for security issues
4. Suggest improvements
5. Write a clear summary

## Checklist
- [ ] Code is readable
- [ ] No obvious bugs
- [ ] Error handling is proper
- [ ] Tests are included
- [ ] Documentation is clear

## Common Issues to Look For
- Null pointer exceptions
- Resource leaks
- Race conditions
- SQL injection
- XSS vulnerabilities
- Infinite loops
- Off-by-one errors
